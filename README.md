# 不動産大家AIチーム

> Claude Code で動く「不動産賃貸業専門の仮想AIチーム」テンプレート

**7人のAIエージェントが、あなたの不動産賃貸業をチームでサポートします。**
秘書を窓口に、物件開拓・融資・戦略・管理・経理・法務の専門エージェントが連携します。

---

## クイックスタート（3ステップ）

```bash
# 1. リポジトリをクローン（または ZIP を解凍）
git clone https://github.com/ritzcr/realestate-ai-team.git
cd realestate-ai-team

# 2. Claude Code を起動
claude
```

> **ZIP で使いたい場合**: [Releases](https://github.com/ritzcr/realestate-ai-team/releases) から ZIP をダウンロードし、解凍後に `cd realestate-ai-team` → `claude` で同じように動きます。

起動後、`こんにちは` と話しかけてみてください。秘書「しおり」がチームを紹介します。

## チーム構成

| ポジション | キャラ | 役割 | フォルダ |
|-----------|--------|------|---------|
| 🎯 **司令塔** | **しおり**（秘書） | 全タスクの窓口・振り分け・TODO管理 | `secretary/` |
| ⚔️ **オフェンス** | **ハンター**（物件開拓＆リサーチ） | 楽待/健美家チェック・指値判断・エリア分析・賃料相場 | `property/` |
| ⚔️ **オフェンス** | **バンカー**（融資担当） | 金融機関プロファイル・融資戦略・銀行アタック資料 | `banking/` |
| ⚔️ **オフェンス** | **軍師**（戦略・参謀） | 投資判断・ポートフォリオ最適化・出口戦略 | `strategy/` |
| 🛡️ **ディフェンス** | **番頭**（賃貸管理） | 入居者対応・修繕判断・管理会社折衝・空室対策 | `management/` |
| 🛡️ **ディフェンス** | **帳簿番**（経理） | 確定申告・月次収支・経費仕訳・収支シミュレーション | `finance/` |
| 🛡️ **ディフェンス** | **奉行**（法務） | 契約書レビュー・滞納対応・敷金返還・立退き交渉 | `legal/` |

不動産賃貸業は野球と同じ。**攻め（事業拡大）** と **守り（資産防衛）** を7人で分担します。

## カスタマイズガイド

### オーナープロフィールを書く（最初にやること）

`CLAUDE.md` を開き、**オーナープロフィール** の欄に自分の情報を記入してください。
保有物件・投資エリア・取引金融機関などを書くと、チームがあなた専用のアドバイスをします。

### キャラ名・口調を変える

`CLAUDE.md` と `.claude/rules/departments.md` の対応行を書き換えるだけです。

### 部署を追加する

1. フォルダを作成（例: `tax-advisor/`）
2. `tax-advisor/CLAUDE.md` を書く（役割・ルール・フォルダ構成・判断軸の4セクション）
3. ルート `CLAUDE.md` の組織図に追記

### 不要な部署を削除する

フォルダごと削除して、ルート `CLAUDE.md` の組織図から該当行を消してください。

## FAQ

**Q: 既存の CLAUDE.md がある場合は？**
A: 上書きしないこと。別フォルダに展開して `claude` を起動するか、自分の CLAUDE.md に組織図セクションを追記してください。プロジェクトレベルの CLAUDE.md はグローバルより優先されます。

**Q: インターネット接続が必要ですか？**
A: このパッケージはローカルファイル操作のみで完結します。MCP 接続（外部 API）は不要です。

**Q: サンプルデータは実在の情報ですか？**
A: すべて架空のデータです。実際の物件・銀行・人物とは一切関係ありません。

**Q: 個人情報を入力しても大丈夫ですか？**
A: ローカル環境で動作します。ただし `.gitignore` で `*-personal.md` などを除外し、実際の入居者情報・銀行担当者名を公開リポジトリにコミットしないよう注意してください。
